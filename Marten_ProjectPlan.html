Delicountr

A web student help queue

Efficient and effective solution for organizing student requests for help

Goals:

  • Act as a final project for WHS Intro to CS class
  • Introduce full stack development and the skills/organization needed
  • Create a real web application to be used by teachers at WHS

Background:

  • This project stemmed from an existing system where students could physically take a number from a deli ticket dispenser when a teacher was swamped with requests for help. This system was effective in organizing these students into a queue, but presents a problem when two teachers are available for help - which makes it very difficult to tell which is the next number.

Technologies

  • Backend
  • LAMP Full stack- Linux, Apache, MySQL, PHP
  • Frontend
  • Languages: HTML5, CSS3, Javascript AJAX
  • Frameworks: Bootstrap, Materialize, jQuery

Features:

  • Simplistic UI
  • Responsive mobile design
  • Teacher
  • Multiple at the same time
  • Navigation menu
  • Serving: Get next number/name/description
  • Viewing: See the queue and expand for description
  • Editing: Clear the queue or delete select students
  • Students
  • Enter the queue by entering their name and (optionally) description of issue
  • Displays what number is currently being served//how many are in line
  • Editing name and description
  • Exit the queue

Screens

  • Index - Take a ticket
  • Enter Queue Pin (future)
  • /teacher Index- Queue Serving
  • Queue Viewing
  • Queue Editing
